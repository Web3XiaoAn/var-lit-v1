# Var-Lit V1 当前策略说明

`adaptive-median-v6` 是 V1 内部公式和冻结持仓上下文的版本标识；公开产品版本仍为
Var-Lit V1。

## 概率门槛

v6继续使用完整的5m、30m、1h实时窗口，但不再在策略候选外额外增加固定开仓门槛。
每个窗口直接计算经验中位数与Q80，并以既有权重估计中位数到Q80的上侧分位跨度：

```text
上侧跨度 = 0.25 × (5m Q80 - 5m中位数)
         + 0.45 × (30m Q80 - 30m中位数)
         + 0.30 × (1h Q80 - 1h中位数)

开仓门槛 = max(5m中位数, 30m中位数, 1h中位数)
         + (58 - 50) / (80 - 50) × 上侧跨度
```

这相当于用三个实时经验分布在中位数至Q80区间内插值到目标Q58。市场分布变宽时门槛
自然上移，变窄时自然回落；不存在独立的固定`0.25bps`附加门。方向均衡和历史成交磨损
只用于诊断，不反馈成越来越高的信号门槛。

## 执行与平仓

- 500U参考深度和实际Firm金额必须越过同一个冻结概率门槛。
- 候选超过`3 × 30m MAD`的异常尖峰仍不参与开仓。
- 最近5秒同方向振幅超过3bps时不在剧烈跳动中开仓。
- 持仓30分钟前，只有当前整轮预估收益大于0且连续2秒保持在0磨损线上方，才可提前
  平仓；断流超过1秒或跌破0会重置连续计时。
- 持仓满30分钟后直接启用配置的基础磨损下限。持仓满1小时后，常规平仓下限在基础值
  上额外放宽`0.2bps`；之后每满10分钟再放宽
  `0.2bps`。金额按实际开仓金额等比例缩放；该阶梯只属于当前持仓，新一轮从基础下限
  重新计算。
- 普通平仓的Lighter腿先用最新深度、IOC和冻结经济限价尝试三次。仍无法成交时，
  自动转为reduce-only恢复单完成对冲，不因一次价格偏离永久暂停自动化。
- 订单结果不明确、账户仓位不一致或无法确定恢复方向时仍暂停并等待人工对账；这些是
  账户安全边界，不是磨损控制手段。

## 数据边界

运行时内存只保留最新一小时。长期行情、执行追踪、订单指标和完整轮次由外部SQLite
研究库按内容哈希去重保存。服务器使用独立低优先级采集服务，数据库写入不进入交易
主循环。

开仓还经过独立的`execution-survival-v2`执行存活校验。它使用按方向归一化的短期波动、
1bps深度不平衡、book/trade flow和microprice，判断当前基差余量能否覆盖本次随机执行
窗口内的不利变化。失败只放弃当前机会，不抬高三窗门槛、不暂停后续自动判断。

开仓的 Lighter 腿先按冻结经济限价进行三次受保护 IOC。若仍未完成，只对剩余数量追加
一次`open-hedge-recovery-v1`恢复 IOC。恢复单忽略已经过时的冻结经济限价，但仍要求最新
完整盘口、有效数据年龄，并严格受配置 IOC 滑点上限约束。恢复仍失败时才平掉 Var，避免
留下单边敞口；该变化不作用于平仓流程。
