# Var-Lit V1 当前策略说明

`adaptive-median-v5` 是 V1 内部模型与存量状态的兼容标识；公开产品版本为 Var-Lit V1。

v5继续使用5m、30m、1h三个完整实时窗口，但把v4固定的 `0.10bps` 缓冲改为轻量波动自适应：

```text
开仓门槛 = max(5m中位数, 30m中位数, 1h中位数)
           + max(0.05bps, 0.20 × 30m MAD)
```

方向均衡统计和历史成交磨损仍用于诊断与复盘，不会反馈进滚动信号门槛。第一次完整一小时样本就绪后立即激活参数；以后每个合格的一分钟候选直接更新epoch，在途订单始终使用其已冻结epoch。

## 开仓质量保护

- 500U参考深度与实际Firm金额都必须越过同一个冻结门槛。
- 候选超过门槛 `3 × 30m MAD` 时视为异常尖峰，不开仓。
- 最近5秒同方向参考价差振幅超过 `3bps`，或5秒窗口尚未形成时，不开仓。
- Firm金额必须位于“目标金额±1U”；模板中的金额不会被脚本改写。
- 最终Firm Guard对Lighter开仓腿加入固定预留，并把最差允许价格绑定到IOC；不增加网络请求，也不把历史坏成交反馈成越来越高的信号门槛。

## 平仓质量保护

真实平仓继续使用最终双边经济价格和原有分段整轮底线。持仓不足30分钟且满足平仓条件时，如果最近5秒平仓方向振幅超过 `4bps`，最多延后2秒等待波动收敛；超过2秒不再因为该短窗门控继续延后，避免风险保护反过来阻塞退出。

## 数据边界

运行时内存只保留最新一小时。`strategy_market_samples.jsonl` 至少保留61分钟，并每9分钟压缩一次，物理跨度最多约70分钟。`tools/analyze_strategy_samples.py` 因此按运行时的一小时覆盖、每秒至少0.10个有效样本、最大有效间隔小于60秒来判定 `qualified_runtime_1h`。更长历史由项目外部研究数据库承担，不要求运行文件保留4小时。

## 扩展与虚拟显示运行

配套扩展构建为 `var-lit-v1`，manifest版本 `1.0.0`。扩展仅申请Variational站点、调试器、标签页和存储权限，不使用 `<all_urls>` 或 `activeTab`。结构模板可以跨重启保存；授权头只进入 `chrome.storage.session`，旧缓存中的授权会在加载时清除。

自动启动默认关闭。启用后，Chrome启动会只绑定现有Variational永续合约页；没有目标页时创建BTC永续合约页。标签页关闭、跳离Var交易页或调试器断开后使用有上限的退避重连。服务器使用普通Chrome和Xvfb虚拟显示器；安装Chrome、安装并解锁OKX钱包、登录Variational与钱包签名仍由账户持有人本人完成。
