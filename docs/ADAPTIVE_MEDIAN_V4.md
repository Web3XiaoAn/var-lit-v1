# adaptive-median-v4 连续实盘修订

v4使用三窗口中位数入场、一分钟参数刷新、Firm Quote/Commit、Lighter IOC对冲和精确数量平仓。执行模式为 `observe/live`：`live` 连续开平，不使用单轮令牌，亏损只记录指标，不会停止后续开仓。订单结果不明确、对冲失败或账户不一致仍会安全暂停。

## 方向均衡

基础方向门槛不变：

```text
Entry_d = max(Median_5m_d, Median_30m_d, Median_1h_d)
          + 0.10bps
```

方向均衡统计仍按15秒合并连续信号。允许保留的独立机会片段数改为：

```text
Allowed_d = max(8, 2 × max(1, OppositeEvents))
Final_d   = Entry_d
```

均衡线继续显示并记录机会分布，但不再成为开仓否决项；30m MAD同样只保留为诊断数据。这样两侧机会仍可复盘，却不会把活跃方向推到极端失真价差。

## Firm金额容差

模板金额不被改写。目标金额可以是任意正数，优先读取捕获模板中的金额字段；旧插件尚未提供该字段时兼容使用其返回的权威Firm金额。Variational返回的Firm价格乘数量必须落在闭区间 `[目标金额-1U, 目标金额+1U]`；例如200U对应199–201U，500U对应499–501U。实际Firm金额用于后续PnL、Lighter数量和风险计算，超出当前目标的±1U只拒绝该次机会，不会永久停机。

Chrome 插件构建为 `2026-07-15-adaptive-median-v4-local4`。平仓 Quote 必须绑定冻结的BTC数量；随后 Commit 可以只带该 Quote 返回的新 `quote_id`，因为数量已经由服务器端Firm Quote冻结。这一修改没有增加下单链路请求数。

## 重启与数据

策略样本格式保持 `adaptive-market-sample-v1`，v4继续读取磁盘中最新一小时合格样本。停止到重启不超过5分钟时，只需接上一帧新鲜行情，不必重新采集一小时。旧版空仓状态中的亏损 `HALTED/REVIEW_REQUIRED` 会被解除；含仓位、待处理意图或真实自动化暂停的状态仍拒绝自动迁移或保持安全暂停。

行情样本只取最新一小时参与计算，磁盘保留约61–70分钟并后台压缩。采样由新鲜Variational报价立即触发，固定1秒定时器只作健康兜底；所有新鲜度和350ms双源偏差校验保持不变，从而避免约3秒的Var更新节奏与固定1秒采样长期相位错位。执行追踪日志采用16MB、3份轮转，重复决策心跳降为10秒一次，均不进入下单热路径。

## 开仓成交保护

三窗口动态门槛负责寻找机会，不再叠加历史成交余量。初筛时500U参考价差与当前金额价差必须达到冻结动态门槛；取得Firm Quote后，用该Quote、精确Firm数量和已经维护的Lighter深度复核同一门槛。

复核只做本地常数时间计算，不新增网络请求；最差可接受经济价格直接进入现有Lighter IOC请求。若校验后价格恶化，IOC拒绝门槛以下的坏价格；价格不变或改善时正常成交。历史磨损样本仍按方向和金额桶保存用于复盘，但不参与正式门槛。

## Lighter低延迟下单门控

策略新开仓只在Lighter专用下单WebSocket处于READY时放行，并在取得Variational Firm Quote后、Commit前再次检查；任一检查失败都会放弃该次开仓，不会先成交Var再主动改走REST。启动的一秒预热窗口只负责快速报告状态，后台仍以250ms节奏重连，恢复后自动开放新开仓，不增加行情判断和Firm链路请求。

该门控只约束新风险敞口。已有仓位平仓、异常恢复以及手动Variational成交后的必要对冲仍保留REST后备，防止因为WebSocket断线而留下未对冲仓位。看板的 `Lighter 下单通道` 行会明确显示 `WS READY`、`REST FALLBACK` 或 `UNAVAILABLE`。

## 平仓预留与零磨损稳定性

平仓预留取封存epoch `0.50bps/leg` 的二分之一，即默认有效 `0.25bps/leg`；策略预检与Firm Quote最终校验使用同一倍率。开仓不再额外叠加该预留。

直接退出仍按分段整轮下界执行。除此之外，运行时只在毛整轮PnL大于0时累计稳定性：连续达到2秒，或最近10秒内多次突破的正区间合计达到2秒，均可发起平仓Firm Quote。行情缺帧或两次有效样本相隔超过1秒会切断连续区间，避免把卡顿误算成稳定。稳定性通道的Firm Quote最终校验仍要求 `实际开仓PnL + Firm平仓PnL >= 0`，屏幕买卖盘不能直接授权成交。

看板将毛估算拆开显示为“此时平仓磨损”“平仓预留”和“当前整轮净估”；净估等于实际开仓PnL加当前平仓PnL再减有效平仓预留。
